---
title: Generated Hooks Reference
description: Technical reference for BAML's auto-generated React hooks
---

BAML automatically generates a type-safe React hook for each BAML function. Each hook follows the naming pattern `use{FunctionName}` and supports both streaming and non-streaming modes.

<CodeBlocks>
```baml title="BAML Function"
function WriteMeAStory(input: string) {
  client openai/gpt-4
  prompt #"
    Tell me a story about {{input}}
  "#
}
```

```typescript title="Example Usage"
import { useWriteMeAStory } from "@/baml_client/react/hooks";

// Basic usage
const { data, error, mutate } = useWriteMeAStory();

// Streaming usage
const { data, streamingData } = useWriteMeAStory({
  stream: true,
  onPartial: (text) => console.log('Story in progress:', text),
});
```
</CodeBlocks>

## HookInput

The hook accepts an optional configuration object. See [Hook Input](./hook-input) for complete details.

<ParamField
  path="stream"
  type="boolean"
>
  Enable streaming mode for real-time updates.
</ParamField>

<ParamField
  path="onPartial"
  type="(response?: PartialReturnType<FunctionName>) => void"
>
  Callback for streaming updates. Only available when streaming is enabled.
</ParamField>

<ParamField
  path="onFinal"
  type="(response?: FinalReturnType<FunctionName>) => void"
>
  Callback when the request completes.
</ParamField>

<ParamField
  path="onError"
  type="(error: Error | BamlValidationError | BamlClientFinishReasonError) => void"
>
  Callback when an error occurs. See [Error Types](../errors/overview).
</ParamField>

## HookOutput

The hook returns an object with the following properties. See [Hook Output](./hook-output) for complete details.

<ParamField
  path="data"
  type="FinalReturnType<FunctionName> | undefined"
>
  The final response data.
</ParamField>

<ParamField
  path="streamingData"
  type="PartialReturnType<FunctionName> | undefined"
>
  Latest streaming update. Only available in streaming mode.
</ParamField>

<ParamField
  path="error"
  type="Error | BamlValidationError | BamlClientFinishReasonError | undefined"
>
  Error information if the request fails. See [Error Types](../errors/overview).
</ParamField>

<ParamField
  path="isPending"
  type="boolean"
>
  True while the request is in progress.
</ParamField>

<ParamField
  path="isSuccess"
  type="boolean"
>
  True if the request completed successfully.
</ParamField>

<ParamField
  path="isError"
  type="boolean"
>
  True if the request failed.
</ParamField>

<ParamField
  path="status"
  type="'idle' | 'pending' | 'success' | 'error'"
>
  Current state of the request.
</ParamField>

<ParamField
  path="mutate"
  type="(...args: Parameters<FunctionName>) => Promise<ReturnType<FunctionName>>"
>
  Function to execute the BAML function.
</ParamField>
