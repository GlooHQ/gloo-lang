title: BAML Generators
---

BAML generators create type-safe client libraries for different languages and frameworks. They transform your BAML functions into idiomatic code for your target platform.

## Overview

Generators provide:
- Type-safe client libraries with full IDE support
- Framework-specific integrations and patterns
- Automatic error handling and type definitions
- Streaming support where applicable
- Test utilities and examples

## Available Generators

### Language Generators

- [TypeScript](/ref/08-generators/01-languages/typescript) - Type-safe client for TypeScript/JavaScript projects
- [Python (Pydantic)](/ref/08-generators/01-languages/python-pydantic) - Python client with Pydantic models
- [Ruby (Sorbet)](/ref/08-generators/01-languages/ruby-sorbet) - Ruby client with Sorbet type checking
- [OpenAPI (REST)](/ref/08-generators/01-languages/rest-openapi) - Language-agnostic REST API specification

### Framework Generator

- [React/Next.js](/ref/08-generators/02-frameworks/react) - React hooks and server actions for Next.js App Router
  - Type-safe React hooks with streaming support
  - Server Actions for Next.js Server Components
  - Full TypeScript type inference
  - Automatic error handling and validation

More framework generators coming soon.

## Generator Configuration

Configure generators in your BAML project using the `generator` block:

<CodeBlocks>
```baml Basic Configuration
generator my_generator {
  output_type language[/framework]  # The type of generator to use
  output_dir "../relative/path"     # Where to output generated code
  version "x.y.z"                  # Generator version to use
}
```
</CodeBlocks>

### Configuration Options

<ParamField
  path="output_type"
  type="string"
>
  The type of generator to use. Format: `language[/framework]`

  Available options:
  - `typescript` - TypeScript client
  - `typescript/react` - React hooks and server actions
  - `python/pydantic` - Python with Pydantic models
  - `ruby/sorbet` - Ruby with Sorbet types
  - `rest/openapi` - OpenAPI specification
</ParamField>

<ParamField
  path="output_dir"
  type="string"
>
  Path where generated code will be placed, relative to your BAML project root.
</ParamField>

<ParamField
  path="version"
  type="string"
>
  Generator version to use. Each generator follows semantic versioning.
</ParamField>

## Best Practices

1. **Project Structure**
   - Keep generated code separate from source code
   - Use consistent relative paths
   - Follow language/framework conventions

2. **Type Safety**
   - Enable strict type checking
   - Never modify generated files
   - Create typed wrappers for customization

3. **Error Handling**
   - Use appropriate error handling for your language
   - Handle validation errors appropriately
   - Implement proper cleanup for streaming
